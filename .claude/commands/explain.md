# 파일 분석 및 설명

분석할 파일: $ARGUMENTS (없으면 현재 열려있는 파일)

## 분석 요청사항

다음 내용을 포함하여 파일을 분석해주세요:

### 1. 프로젝트 내 역할
- 이 파일이 프로젝트 구조에서 어떤 위치에 있는지
- 다른 파일들과의 관계 (import/export, 의존성)
- 이 파일이 담당하는 책임 범위

### 2. 주요 기능
- 파일 내 정의된 함수, 컴포넌트, 클래스, 타입 등
- 각 요소의 목적과 동작 방식
- 외부로 노출되는 API (export)

### 3. 위험 요소 분석
다음 항목들을 검토해주세요:
- **보안 취약점**: XSS, SQL Injection, 인증/인가 문제 등
- **성능 이슈**: 불필요한 리렌더링, 메모리 누수, 무한 루프 가능성
- **코드 품질**: 하드코딩된 값, 에러 핸들링 부재, 타입 안전성 문제
- **유지보수성**: 과도한 복잡도, 매직 넘버, 문서화 부족

### 4. 개선 제안 (있다면)
- 발견된 문제에 대한 구체적인 해결 방안
- 코드 품질 향상을 위한 리팩토링 제안

---

**참고**: 파일이 지정되지 않은 경우, 현재 IDE에서 열려있는 파일을 기준으로 분석합니다.
